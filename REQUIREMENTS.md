# Open Skills GUI v1 需求基线

本文件记录 v1 已确认的产品、架构、数据和发布边界。后续修改数据结构或兼容旧数据前，必须重新确认方案。

## 1. 产品范围与固定版本

- 产品名为 **Open Skills GUI**，应用标识为 `io.github.fb208.openskillsgui`，使用 AGPL-3.0。
- 仅支持 Windows 10 22H2（19045）和 Windows 11 原生 x64，仅管理全局 Skill。
- 界面仅提供简体中文，不显示或解析面向用户的 Skill 介绍。
- 使用 Neutralinojs 6.7.0、客户端库 6.5.0、构建工具 11.7.1、Preact 与 TypeScript。
- 固定调用 `skills@1.5.19`；私有运行环境固定为 Node 24.18.0 x64 与 MinGit 2.55.0.2 x64，Node 最低版本为 22.20.0，Git 最低版本为 2.40。
- 安装程序固定使用 Inno Setup 6.7.3。
- 安装目录为 `%LOCALAPPDATA%\Programs\OpenSkillsGUI`，数据目录为 `%LOCALAPPDATA%\OpenSkillsGUI`。

## 2. 架构与安全边界

- 界面采用 Preact、TypeScript 和 Vite，包含“已安装”“搜索”“设置”三个主页面。
- Skill、文件、网络下载和进程管理由单文件 Node 扩展完成；界面只通过带请求编号的扩展接口调用。无 Node 时，界面仅可启动固定的运行环境引导脚本。
- Neutralinojs 使用一次性令牌、严格内容安全策略和最小原生权限。
- 外部命令必须使用绝对可执行文件路径和参数数组，不把用户输入拼接为命令行字符串。
- 所有写操作串行执行；远端来源检查最多并发 3 组。
- 删除、移动和建立目录联接前必须验证允许根目录、直接子目录关系及目录链接类型。
- 日志仅保存在本机，隐藏用户主目录，不记录敏感环境变量，不上传遥测或崩溃数据。

## 3. 数据模型与目录

- 状态使用 UTF-8 JSON，固定 `schemaVersion: 1`，通过临时文件和原子替换保存。
- 未知或损坏的数据版本必须拒绝加载；v1 不设计旧配置、旧数据迁移或回退。
- Skill 记录包含稳定身份、名称、来源、分支、来源子路径、目标 Agent、启用状态、托管状态、内容哈希、备注和路径。
- 状态仅允许“启用”“禁用”“已卸载”；托管状态与启用状态独立。
- 已知来源以规范化来源、分支、子路径和名称生成稳定身份；未知来源使用本工具生成的 UUID。
- 卸载 Skill 后记录转为“已卸载”并从普通列表隐藏，但备注、来源和创建时间保留。
- `data`、`workspace\disabled`、`logs` 是保留的用户数据；`runtime`、`cache`、`updates` 是可清理的运行文件。

## 4. 运行环境

- 每次启动先检查本工具私有环境，再检查系统 Node、配套 npx 和 Git。
- 环境缺失时询问用户；同意后从 Node.js 和 Git for Windows 官方固定地址下载 x64 ZIP，校验固定 SHA-256 后解压到私有 `runtime`，不修改系统 PATH。
- npm 使用独立缓存；所有 skills 调用固定为 `skills@1.5.19`，并设置 `DISABLE_TELEMETRY=1` 与 `DO_NOT_TRACK=1`。
- 环境安装成功后重启应用；失败时保留设置页和中文错误详情，不开放 Skill 管理。
- 安装程序必须检测 WebView2。缺失时先征得同意，再运行微软官方安装器；拒绝或安装失败则终止应用安装。

## 5. 扫描、接管与日常管理

- 扫描 `%USERPROFILE%\.agents\skills`、`.claude\skills`、`.codeium\windsurf\skills`，并结合 `skills list -g --json` 和官方锁文件识别来源。
- 通用目录覆盖 Codex、OpenCode、Cursor、Gemini CLI、GitHub Copilot 等；Claude Code 和 Windsurf 使用指向通用实体目录的目录联接。
- 首次发现旧 Skill 时询问是否整体接管。拒绝后显示“未托管”，仍允许查看、搜索和备注；启停、更新或卸载前必须接管。
- 接管按“复制到临时区—校验哈希—切换实体和联接—写入状态”执行。同名且内容相同的副本合并；内容不同必须由用户选择保留项或跳过，绝不静默覆盖。
- 未知来源按 Skill 名称精确搜索；无结果显示“无法更新”，唯一或多个候选均需用户确认后绑定。
- 远端搜索调用固定版 `skills find`，输入防抖并可取消过期搜索；只展示名称、来源、安装量和安装状态，最多 6 项。
- 搜索输出格式变化时明确失败，不调用非公开接口。
- 安装使用官方 `skills add` 完成通用实体安装，再为所选 Claude Code、Windsurf 目标建立目录联接。
- 已安装列表按名称、来源和备注进行本地模糊匹配。
- 禁用时移除本工具管理的目录联接，将实体原子移动到 `workspace\disabled`；启用时反向执行。
- 启用和禁用状态都允许更新和卸载。
- 文件被占用时通过 Windows Restart Manager 展示进程名与 PID；用户确认后用 `taskkill /T /F` 终止进程树并重试。权限不足时明确失败，不绕过 Windows 权限。

## 6. Skill 更新

- 检查更新按来源分组，在隔离的临时用户目录中调用固定版 `skills add` 获取远端内容，不触碰现有 Skill。
- 仅 skills.sh 可解析的公开 GitHub 来源支持更新；本地、私有、Git 或未知来源显示“无法更新”。
- 使用“上次安装内容哈希、当前本地哈希、远端哈希”分类：
  - 本地未改、远端变化：可更新；
  - 本地变化、远端未变：本地已修改；
  - 本地和远端都变化：更新冲突。
- 状态统一返回“已是最新、有可用更新、本地已修改、更新冲突、无法更新、检查失败”。
- 批量更新跳过本地修改和冲突；单个冲突更新必须二次确认后才允许远端覆盖。
- 启用 Skill 由官方命令更新并重新校验实体和联接；禁用 Skill 在隔离目录取得新版后原子替换工作区副本，更新过程中不得重新启用。
- 单个和批量操作必须逐项显示最终结果。

## 7. 软件更新、安装与发布

- 每次启动异步请求 `FB208/open-skills-gui` 最新正式 GitHub Release，并提供手动检查。
- 自动检查失败不打扰用户；手动检查失败显示错误。
- 仅当正式稳定语义版本更高时显示更新和 Release 说明。
- 用户确认后下载固定名称 `OpenSkillsGUI-Setup-x64.exe`，校验 Release 资源 SHA-256，再启动升级辅助脚本。
- 辅助脚本等待当前应用退出，以 Inno Setup 静默覆盖安装并重新启动；禁止无提示后台升级、强制升级或降级。
- 安装程序使用固定 AppId、当前用户安装和无 UAC 模式。
- 卸载只删除程序、私有运行环境、缓存和更新临时文件，保留用户数据、备注、Skill 和目录联接。
- 最终只发布 `OpenSkillsGUI-Setup-x64.exe`，不生成便携包，不签名；文档明确提示 SmartScreen 可能显示未知发布者。
- GitHub Actions 在 `windows-latest` 执行依赖审计、格式检查、类型检查、测试、Neutralinojs 构建、Inno Setup 编译和安装/卸载冒烟。
- PR 和主分支上传安装程序 CI 工件；`vX.Y.Z` 标签必须与应用版本一致，并创建只含安装程序的正式 Release。

## 8. 验收与明确不做

- 自动测试覆盖模糊匹配、版本、来源规范化、状态、哈希、路径边界、CLI 解析、事务恢复、软件更新响应和 Windows 目录生命周期。
- Windows 集成测试覆盖扫描、同名冲突、接管、目录联接、启停、两种状态卸载、备注保留、文件占用和中断恢复。
- 受控真实 CLI 冒烟测试验证公开 Skill 搜索、隔离安装和更新检查；干净 Windows 10/11 x64 虚拟机执行最终人工验收。
- 不支持项目级 Skill、Windows x86、ARM64 或其他系统。
- 不提供 Skill 介绍、代码签名、便携包、系统托盘、开机启动、后台服务、私有源鉴权、遥测、崩溃上报或静默强制升级。
- GitHub 更新检查、Skill 搜索、安装和更新是用户使用功能所需的网络访问，不视为遥测。
